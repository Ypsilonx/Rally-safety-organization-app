# Security Policy

Dekujeme, ze pomahate drzet projekt bezpecny.

## Supported Versions

Aktivne podporovana je aktualni vyvojova vetev `main`.

## Reporting a Vulnerability

- Nezverejnujte bezpecnostni chyby verejne v issues.
- Pokud je v repozitari dostupne private vulnerability reporting (GitHub Security Advisories), pouzijte ho.
- Pokud private reporting neni dostupny, vytvorte issue s prefixem `SECURITY:` bez citlivych detailu a maintainer s vami domluvi bezpecny kanal.

## Co cekat po nahlaseni

- Potvrzeni prijeti do 72 hodin.
- Prubezny update o stavu analyzy.
- Po oprave koordinovane zverejneni detailu.
